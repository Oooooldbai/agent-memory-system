# 消防队模式（Firefighter Mode）

> 短生命周期子Agent + 异步调度 + 检查点驱动

## 核心思想

**传统（阻塞）**：派Agent → 等完成 → 收结果 → 再派下一个
**消防队（并行）**：拆解 → 批量触发 → 立即返回 → 异步检查检查点

## 三角色

| 角色 | 职能 | 行为 |
|------|------|------|
| **监控中心** | 任务拆解、看进度、异常重派 | 深度思考，不阻塞 |
| **调度中心** | 生成子任务、批量触发、立即汇报 | 快速分发，不等待 |
| **消防员** | 单批执行(<5min)、写文件、标检查点、退出 | 短命，立即死亡 |

## 执行规范

### 消防员5步

```
1. 读参数  2. 执行  3. 写结果  4. 标检查点  5. 退出
```

禁止：等待其他Agent / 汇报进度 / 轮询状态 / 运行>5min

### 监控中心流程

```
拆解为批次 → 触发调度 → 不等待 → 定期查检查点 → 缺失重派 → 汇总
```

### 调度中心流程

```
接收任务 → 拆子任务 → 批量spawn（不等待）→ 汇报"已触发X个" → 退出
```

## 检查点格式

```json
{
  "task_id": "batch_N_part_M",
  "status": "completed|failed|pending",
  "output_files": [],
  "timestamp": "ISO"
}
```

## 适用判断

✅ 数据采集 / 批量处理 / 并行分析
❌ 深度推理 / 强顺序链式 / 实时交互

## Token预算

| 环节 | 消耗 |
|------|------|
| 消防员单次 | 1-3k |
| 调度批量触发 | 0.5-1k |
| 监控查进度 | 0.2-0.5k |
| **10批总计** | **15-25k（比阻塞省50%+）** |

## 确认→固化→验证 闭环

### 触发条件
用户说"确认/就这么定了" / 同话题>3轮 / 涉及配置变更

### 四步

1. **确认** — 明确变更+影响范围 → 检查点 `impact.json`
2. **固化** — 更新配置+记忆文档 → 检查点 `config.json`
3. **验证** — 立即测试一次 → 检查点 `verify.json`
4. **归档** — 合并检查点，标记完成

### 自检（任一未做=未固化）
- [ ] 配置更新了？  [ ] 检查点创建了？  [ ] 记忆更新了？  [ ] 验证执行了？

### 强制同步场景
新协作机制 / 新增配置 / 新增账号 / 修改规则文件 / 新文件结构
→ 必须立即写入 MEMORY.md + index.md，不等"收工"

### 检查点强制创建
新机制/新配置/规则修改 → `checkpoints/memory/YYYY-MM-DD-{name}.json`

```json
{
  "checkpoint_type": "mechanism_created | config_added | rule_updated",
  "description": "做了什么",
  "files": {},
  "synced_to": { "MEMORY.md": false, "index.md": false },
  "lesson_learned": ""
}
```
